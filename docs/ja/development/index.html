開発とサポート¶

ライセンス¶

QFieldは、GNU Public License（GPL）バージョン2以降でリリースされています。 このライセンスの下でQFieldを開発するということは、ソースコードをチェックおよび変更できることを意味し、無償かつ自由に変更可能なQGISベースのフィールドデータコレクションアプリに常にアクセスできることを保証します。

You can find the sourcecode on github

サポート¶

You can ask your questions on gis.stackexchange

You can use the user mailing list

商用サポートも行っています。 こちら ご連絡ください。

QFieldを成長させ、自分のものにしよう¶

これらは、素晴らしいユーザーエクスペリエンスと効果的なワークフローに非常に役立つと思われる機能のリストです。

編集ウィジェット¶

編集ウィジェットは、効率的なフィールドワークをするための中核をなす部分です。 タッチデバイスで属性データを入力することは、最も時間がかかる業務の1つであり、このタスクをできるだけ簡単に保つことが仕事を楽にするためのカギとなります。

いくつかの重要な 編集ウィジェットはいつでもお使いいただけます。 しかしさらに多くのことが QGISの活用例 からご利用いただけます。

凡例¶

マップのコンテンツを適切に解釈するには、凡例が非常に有用なツールとなります。 ユーザーに、色、形、線、その他のビジュアルがどういった意味かを理解するために必要な情報を提供しています。

また、個々のレイヤーの追加メタデータにアクセスしたり、編集したいレイヤーを選択するために使用することもできます。

オンライン/オフライン間での同期¶

データ収集は、QFieldが大いに役立つ分野の1つです。 データの最終的な保存先は通常、デバイス自体ではなく、データベース上のどこかになるでしょう。 QFieldはデータベースに対して直接操作することができます。但し、現場においてはインターネットアクセスがいつも使えるとは限りませんので、デスクトップマシン上で手動での同期処理を行えます。

インターネットが利用可能な場合データがサーバーとデバイス間で自動的に同期され、インターネットが利用できない場合は常にデータをローカルのデバイス上にキャッシュされることは素晴らしいことです。

これは、 GeoGIG またはデータベースなどの他のサーバーコンポーネントと組み合わせることができます。

座標精度の改善¶

コンピュータは、小数（浮動小数点数）を内部に格納するために、主に2つの異なるデータ型で動作します。すなわち、 単精度 と 倍精度 の2つです。 単精度では、数字の最初の6桁のみが重要であり、信頼できます。倍精度では、最初の15桁が信頼できます。 これには、すべての数字（小数点の前後）が考慮されます。

座標を意識して作業する場合、高精度なデータが必要な場合にこれが非常に重要になります。 座標の有効桁数が6桁しかないということは、最大精度がおよそ1メートルであることを意味しています。但しこれはCRSと地表上の位置によって異なる場合があります。

QFieldの処理のいくつかでは単精度によって座標を処理しています。これを完全な精度で処理できるようにすることは正確な測位を行うに大変重要な改善点であるといえます。

GPSのステータス情報を属性に保存¶

測位機器は緯度と経度以外の多くの情報を取得できます。それらの情報と通して測位可能な座標精度を推測することができます。この情報は例えば受信した衛星の数などの変数から計算したり、RTKステータスなどのパラメータからなるフラグであったりします。

ジオメトリの編集¶

QFieldを使って、新しいジオメトリのデジタイジング、既存の属性データの編集、またはフィーチャの削除を行えます。 但し既存のジオメトリを編集する機能は現在ありません。

既存ジオメトリの編集とは、ポイントの移動、他のフィーチャの移動、ノードの編集、エッジの移動など、さまざまな部分から構成されるでしょう。

検索機能¶

検索機能を使ってフィーチャの検索をすることは仕事を効率的に行うのにとても役に立つでしょう。一般的には検索機能は二つのカテゴリーに分類されます：

属性による検索

検索結果は、プロジェクト内のレイヤー上のフィーチャの属性に基づいて提供されます。 検索用のローカルデータベースに対しての利用が可能です。 内部のデータは、作業中のカスタマイズされたデータに基づいているか、検索対象の属性を持つフィーチャのレイヤーが直接検索されます。

ジオコーディング

検索結果は、実際の住所に基づいて提供されます。 `Nominatim <http://nominatim.openstreetmap.org/>`_のようなサービスは、Webインターフェースを介してアクセスし、実際の住所を地図上の場所に変換できます。 リンクを試してください。

外付け測定機器¶

センサー付きの外付け機器を使っていますか？ 気圧、温度、勾配、一酸化炭素濃度、それらの名前を付ける。外付け機器から情報を読み取って手動で再度入力する必要がないに越したことはないでしょう。 外付け測定機器をQFieldに直接接続するためのサポートを追加することにより、お客様の業務をより簡単にすることができます。

音声の録音¶

タッチスクリーンを使用してフィールドに出たときに、どのように多くのテキストを入力するのか疑問に思ったことはありませんか？

録音はしないで!

画像の統合と同じように音声を統合できる可能性があるのは素晴らしいことです。 オフィスに戻ったら、通常のキーボードで情報を入力するだけで済みます。

クリックでQGSファイルを開く¶

ファイルマネージャーでファイルをクリックしてQFieldでQGISプロジェクトを開くだけです。
