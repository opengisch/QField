Налаштування векторних шарів¶

Більшість настройок, доступних в QGIS, підтримуються QField без додаткових зусиль.

Стилі¶

Всі налаштування стилів з QGIS напряму підтримуються QField. Включаючи всі типи відображдення на кшалт градуйованого, категорій, правил, 2.5D та стилів, які залежать від даних.

Вираз для відображення¶

В QField об’єкти ідентифікуються за іменем. Вираз, що генерує це ім’я можна налаштувати. Для цього відкрийте таблицю атрибутів в QGIS та перейдіть у режим форми. Виберіть відповідне поле або вираз зі списку. Починаючи з QGIS 3, це можна зробити напряму на сторінці Vector Layer Properties ‣ Display.

The display expression is also used to search in layers.

Вираз для відображення буде використовуватися для відображення обїектів під час їх ідентифікації в QField.

Read only, non-identifiable and searchable layers¶

Деякі шари в проекті мають лише візуальну функцію. Такі шари не повинні потрапляти у список коли користувач виконує ідентифікацію об’єктів.

Інші шари слугують джерелом інформації та не повинні змінюватися користувачем. Ми передбачили можливість захисту шарів від редагування атрибутів або створення чи видалення нових об’єктів.

It is also possible to configure which layers are searchable.

To configure identify, readonly and searchable behavior, go to the Project ‣ Project Properties ‣ Identify Layers page and activate the checkboxes to match your desired behavior.

Configuration of layers that will not be identifiable, not modifiable and/or not searchable.

Інтерактивний редактор форм¶

Повна підтримка груп та вкладок інтерактивного редактора форм QGIS.

Умовна видимість¶

Групи можуть бути приховані в залежності від результатів виразу. Це може бути корисним, коли певні атрибути необхідні лише в певних умовах. Наприклад, під час картографування дерев. Деякі з них можуть мати захворювання і у вас є список можливих захворювань. Щоб не перевантажувати форму багатьма полями, зробіть групу та налаштуйте вираз видимості для групи “diseases”. Додайте поле-перемикач “desease”. Список захворювань буде видимий лише коли перемикач активний.

Настройки групи, яка буде відображатися лише коли прапорець “виміри зроблено” активовано.

Обмеження¶

На атрибути можна накладати обмеження. Обмеження — це вирази, які повинні повертати True (або 1) щоб вміст форми можна було зберегти. Обмеження налаштовуються дл кожного поля окремо, також можна вказати текст, який буде показаний користувачу, якщо обмеження не виконується.

Настройка обмежень діапазону

Приклади використання¶

Ви не можете ввести висоту більшу за найвищу вершину країни.

"elevation" < 5000

Необхідно заповнити ідентифікатор.

"identifier" IS NOT NULL

Типові значення¶

Поля можуть мати типові значення. Типові значення вставляються у форму атрибутів під час оцифровки об’єкта. Користувач їх бачить та може змінити, якщо поле може редагуватися. Типові значення можуть базуватися на значеннях геометрії чи виразів.

Настройка дати як величини за замовчуванням

Additional variables¶

In QField, there are some additional variables available regarding the current positioning information.

  • position_coordinate (Geometry in WGS84!) : Lon, Lat, Altitude as delivered by the sensor. Is also available when the crosshair is not snapped to the sensor.
  • position_timestamp (DateTime) : Timestamp of the position information
  • position_direction (float) : Direction in degrees from true north.
  • position_ground_speed (float) : Groundspeed in meters / second.
  • position_magnetic_variation (float) : The angle between the horizontal component of the magnetic field and true north, in degrees. Also known as magnetic declination. A positive value indicates a clockwise direction from true north and a negative value indicates a counter-clockwise direction.
  • position_horizontal_accuracy (float) : The horizontal accuracy of the coordinate (in meters).
  • position_vertical_accuracy (float) : The vertical accuracy of the coordinate (in meters).
  • position_vertical_speed (float) : The value of the vertical speed in meters per second.

Приклади використання¶

Insert positioning information as variable:

@position_horizontal_accuracy

Вставити поточну дату та час:

now()

Вставити довжину оцифрованої лінії:

length($geometry)

Налаштувати глобальні змінні на пристрої та вставити їх.

@operator_name

Якщо ви хочете призначити об’єктам код регіону в залежності від того, де сами вони були додані, використовуйте агрегатну функцію:

aggregate( layer:='regions', aggregate:='max', expression:="code", filter:=intersects( $geometry, geometry( @parent ) ) )

Редагування¶

Атрибут редагування поля має значення.

Запам’ятовувати останні значення¶

QField пропонує більш гнучке налаштування використання останніх внесених значень та ігнорує настройку QGIS.

Прилипання¶

Під час оцифровки нових об’єктів, вершини можуть приклеюватися до наявних об’єктів.

Всі настройки знаходяться в Project ‣ Snapping Settings (Settings ‣ Snapping Settings у QGIS 2.x).

Типи прилипання¶

можна приклеювати нові точки

  • тільки до вузлів наявних об’єктів

  • тільки до сегментів наявних об’єктів

  • до вершин та сегментів наявних об’єктів

Прилипання до шарів¶

Також можна приклеюватися до одного чи декількох шарів.

Поріг приклеювання¶

Поріг приклеювання можна задавати у пікселях чи одиницях мапи.

Практично у всіх випадках в якості одиниць слід вибирати пікселі. Ми радимо встановити величину порогу у 20 пікселів.
