Налаштування векторних шарів¶

Більшість настройок, доступних в QGIS, підтримуються QField без додаткових зусиль.

Стилі¶

Всі налаштування стилів з QGIS напряму підтримуються QField. Включаючи всі типи відображдення на кшалт градуйованого, категорій, правил, 2.5D та стилів, які залежать від даних.

Вираз для відображення¶

В QField об’єкти ідентифікуються за іменем. Вираз, що генерує це ім’я можна налаштувати. Для цього відкрийте таблицю атрибутів в QGIS та перейдіть у режим форми. Виберіть відповідне поле або вираз зі списку. Починаючи з QGIS 3, це можна зробити напряму на сторінці Vector Layer Properties ‣ Display.

The display expression is also used to search in layers.

Вираз для відображення буде використовуватися для відображення обїектів під час їх ідентифікації в QField.

Шари лише для читання, не для визначення та пошуку¶

Деякі шари в проекті мають лише візуальну функцію. Такі шари не повинні потрапляти у список коли користувач виконує ідентифікацію об’єктів.

Інші шари слугують джерелом інформації та не повинні змінюватися користувачем. Ми передбачили можливість захисту шарів від редагування атрибутів або створення чи видалення нових об’єктів.

Також можна визначити які шари будуть доступні для пошуку.

Для налаштування властивостей визначення, читання та пошуку перейдіть до сторінки Проект ‣ Властивості проекту ‣ Визначення шарів та активуйте відповідні прапорці.

Налаштування шарів, які не будуть ідентифікуватись, не змінюватимуться та/чи не шукатимуться.

Форми та поля¶

QField створює форми, схожі на форми QGIS, але не однакові з ними.

Не показувати форму¶

Ця настройка застосовується до форм QField напряму.

Relation Widget¶

For more information refer to the relations chapter of the official QGIS Documentation.

The relation widget shows all the referencing child features in a list. It’s possible to add, delete and open them.

The visual identification of the list entries are done a Display Expression on the child layer.

Key handling

Since the parents primary keys are used as foreign keys on the referencing child features, these primary keys must be save to use even after the synchronization back to the original data. Therefore we recommend to use stable values such as UUIDs for primary keys.

Child features can be added before the parent is finaly saved - means while adding a parent. But it’s blocked if there is no valid primary key on the parent or the constraints are violated. In case the adding of the parent feature is canceled after there have been some childs added already, the childs are deleted by cascade.

Many-To-Many relations

On many-to-many relationships, according to the cardinality setting in the QGIS Vector Layer Properties, on adding, deleting or opening an entry in the list effects directly the child layer (on direct cardinality) or the linking table (on many-to-one cardinality).

The second case is usually used when there are additional relation information (e.g. percentage) in the linking table.

In case of the many-to-one cardinality on many-to-many relations the Display Expression needs to be set on the linking table.

External Resource (photo settings)¶

The widget type «Attachement» is used to store the path to collected photos.

The path should be set to relative. Means the pictures are stored then in a sub-directory of the project, where the link stored in the textfield is pointing to.

In QField the picture is displayed. By clicking the camera symbol the camera is started to update the picture with a new photo.

On synchronization the sub-directory with the pictures has to be copied as well.

Інтерактивний редактор форм¶

Повна підтримка груп та вкладок інтерактивного редактора форм QGIS.

Умовна видимість¶

Групи можуть бути приховані в залежності від результатів виразу. Це може бути корисним, коли певні атрибути необхідні лише в певних умовах. Наприклад, під час картографування дерев. Деякі з них можуть мати захворювання і у вас є список можливих захворювань. Щоб не перевантажувати форму багатьма полями, зробіть групу та налаштуйте вираз видимості для групи «diseases». Додайте поле-перемикач «desease». Список захворювань буде видимий лише коли перемикач активний.

Настройки групи, яка буде відображатися лише коли прапорець «виміри зроблено» активовано.

Обмеження¶

На атрибути можна накладати обмеження. Обмеження — це вирази, які повинні повертати True (або 1) щоб вміст форми можна було зберегти. Обмеження налаштовуються дл кожного поля окремо, також можна вказати текст, який буде показаний користувачу, якщо обмеження не виконується.

Настройка обмежень діапазону

Приклади використання¶

Ви не можете ввести висоту більшу за найвищу вершину країни.

"elevation" < 5000

Необхідно заповнити ідентифікатор.

"identifier" IS NOT NULL

Типові значення¶

Поля можуть мати типові значення. Типові значення вставляються у форму атрибутів під час оцифровки об’єкта. Користувач їх бачить та може змінити, якщо поле може редагуватися. Типові значення можуть базуватися на значеннях геометрії чи виразів.

Настройка дати як величини за замовчуванням

Додаткові змінні¶

У QField доступні додаткові змінні, які стосуються інформації про поточне місцерозташування.

  • position_coordinate (Geometry in WGS84!) : Довгота, широта, висота отримані від сенсора. Доступні також коли перехрестя на карті не прив’язане до сенсора.
  • position_timestamp (DateTime) : Час отримання інформації про місцезнаходження
  • position_direction (float) : Напрям у градусах від істинної півночі.
  • position_ground_speed (float) : Швидкість у метрах за секунду.
  • position_magnetic_variation (float) : Кут між горизонтальною компонентою магнітного поля та істинною північчю у градусах. Також відомий як магнітне відхилення. Додатнє значення вказує на напрямок за годинниковою стрілкою від справжньої півночі, а від’ємне значення вказує напрямок проти годинникової стрілки.
  • position_horizontal_accuracy (float) : Горизонтальна точність для координат (у метрах).
  • position_vertical_accuracy (float) : Вертикальна точність для координат (у метрах).
  • position_vertical_speed (float) : Значення вертикальної швидкості у метрах на секунду.

Приклади використання¶

Введіть інформацію позиціонування як змінну:

@position_horizontal_accuracy

Вставити поточну дату та час:

now()

Вставити довжину оцифрованої лінії:

length($geometry)

Налаштувати глобальні змінні на пристрої та вставити їх.

@operator_name

Якщо ви хочете призначити об’єктам код регіону в залежності від того, де сами вони були додані, використовуйте агрегатну функцію:

aggregate( layer:='regions', aggregate:='max', expression:="code", filter:=intersects( $geometry, geometry( @parent ) ) )

Редагування¶

Атрибут редагування поля має значення.

Запам’ятовувати останні значення¶

QField пропонує більш гнучке налаштування використання останніх внесених значень та ігнорує настройку QGIS.

Прилипання¶

Під час оцифровки нових об’єктів, вершини можуть приклеюватися до наявних об’єктів.

Всі настройки знаходяться в Project ‣ Snapping Settings (Settings ‣ Snapping Settings у QGIS 2.x).

Типи прилипання¶

можна приклеювати нові точки

  • тільки до вузлів наявних об’єктів
  • тільки до сегментів наявних об’єктів
  • до вершин та сегментів наявних об’єктів

Прилипання до шарів¶

Також можна приклеюватися до одного чи декількох шарів.

Поріг приклеювання¶

Поріг приклеювання можна задавати у пікселях чи одиницях мапи.

Практично у всіх випадках в якості одиниць слід вибирати пікселі. Ми радимо встановити величину порогу у 20 пікселів.

3D Координати¶

Пристрої GPS можуть вимірювати висоту над рівнем моря поруч із 2D місцезнаходженням на земній поверхні. Цю інформацію можна зберігати
