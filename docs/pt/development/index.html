Desenvolvimento e suporte¶

Licença¶

QField está sob a Licença Pública GNU (GPL) Versão 2 ou acima. O desenvolvimento do QField sob esta licença significa que pode inspeccionar e modificar o código-fonte e garante que você sempre terá acesso a um aplicativo de recolha de dados de campo baseado no QGIS que seja gratuito e que possa ser modificado livremente.

Pode encontrar o código fonte no github

Faça-o crescer, Faça-o seu¶

Esta é uma lista de funcionalidades que consideramos muito úteis para uma boa experiência do utilizador e um fluxo de trabalho eficaz.

Widgets de edição¶

Os widgets de edição são uma parte essencial do trabalho de campo eficiente. Inserir atributos num dispositivo de toque é um dos trabalhos que leva mais tempo e manter essa tarefa o mais fácil possível é a chave para dar facilitar o trabalho.

Alguns importantes: ref: widgets de edição de campo já estão disponíveis<edit_field_widgets> mas muito mais são possíveis por exemplo no QGIS.

Legenda¶

Para interpretar correctamente o conteúdo de um mapa, uma legenda é uma ferramenta muito útil. Ele fornece ao utilizador o contexto necessário para entender o significado semântico das cores, formas, traços e outras codificações visuais.

Ele também pode ser usado para aceder a metadados adicionais de camadas individuais ou escolher em qual camada trabalhar.

Ligação baseada na sincronização on-/offline¶

A recolha de dados é um dos domínios em que o QField pode ajudar muito. O destino final dos dados normalmente não está no próprio dispositivo, mas num lugar dentro de qualquer base de dados. Actualmente, é possível trabalhar directamente na base de dados, mas como o acesso à rede muitas vezes não é confiável quando se está no campo, ele resumese à sincronização manual num computador desktop.

Seria óptimo ter um processo de sincronização em execução nos dispositivos onde os dados serão sincronizados automaticamente com um servidor quando a rede estiver disponível e os dados do cache localmente sempre que não houver ligação de rede disponível.

Pode ser combinado com GeoGIG ou outros componentes de servidor como uma base de dados.

Precisão de Coordenadas melhorado¶

Os computadores trabalham principalmente com dois tipos de dados diferentes para armazenar decimais (números de ponto flutuante) internamente. Única <https://en.wikipedia.org/wiki/Single-precision_floating-point_format> _ e double <https://en.wikipedia.org/wiki/Double-precision_floating-point_format> _ precisão. Com precisão única, somente os primeiros 6 dígitos de um número são significativos e confiáveis; com precisão dupla, os primeiros 15 dígitos são confiáveis. Isso leva em consideração todos os dígitos (antes e depois do ponto decimal).

Quando se trabalha com coordenadas, isto é muito importante quando uma alta precisão é necessária. Somente ter 6 dígitos significativos significa, frequentemente, ter uma precisão máxima em torno de um metro, mas depende do SRC e da localização na superfície da Terra.

Algumas partes do QField actualmente tratam coordenadas com ponto único de precisão. Seria uma melhoria muito boa poder usar a precisão total de dispositivos de posicionamento precisos, corrigindo essa falha.

Guardar a informação do estado do GPS nos atributos¶

Os dispositivos de Posicionamento oferecem mais informação do que apenas latitude e longitude. Eles também são capazes de estimar a precisão que actualmente são capazes de fornecer. Essa informação pode ser um raio calculado a partir de variáveis como o número de satélites, mas também sinalizadores adicionais para parâmetros como o status do RTK.

Edição de Geometrias¶

O Qfield oferece a digitalização de novas geometrias, editar atributos existentes ou apagar elementos. O que falta é o recurso para editar as geometrias existentes.

Isso pode ser dividido em diferentes partes, como mover pontos, mover outros recursos, editar vértices, mover bordas ...

Funcionalidade de Pesquisa¶

Localizar elementos com a ajuda de um campo de pesquisa é algo que pode realmente ajudar a trabalhar de forma mais eficiente. Em geral, isso pode ser dividido em duas categorias:

Pesquisa baseada no atributo

Os resultados da pesquisa são fornecidos com base nos atributos de elementos em camadas dentro do projecto. Uma base de dados local para pesquisa está disponível. Os dados internos são baseados nos dados personalizados com os quais você trabalha ou as camadas que são consultadas directamente para os elementos com esses atributos.

Geocodificação

Os resultados da pesquisa são fornecidos com base em endereços reais. Serviços como o Nominatim <http://nominatim.openstreetmap.org/> _ podem ser acedidos através de uma interface web e converter endereços do mundo real para locais num mapa. Tente utilizar o link.

Dispositivos externos de medição¶

Está a trabalhar com dispositivos externos com sensores? Pressão do ar, temperatura, inclinação, concentração de monóxido de carbono, diga qual ... Não seria óptimo não ter que ler as informações do seu dispositivo externo e digitá-las manualmente de novo? Gostaríamos muito de facilitar a sua vida adicionando suporte para ligar directamente ao seu dispositivo de medição externa ao QField.

Registar Audio¶

Já imaginou como você escreveria muito texto quando estiver em campo com uma tela sensível ao toque?

Não faça isso!

Seria incrível ter a possibilidade de ter integração de áudio como a integração de imagens. Quando voltar ao escritório, você pode introduzir as informações num teclado normal.

Abrir ficheiros QGIS num clique¶
